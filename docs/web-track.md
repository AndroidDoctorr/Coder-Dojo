<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">What you will do in this track</h2>

In this track, you will:
- build simple web pages
- style them with CSS
- add interactivity with JavaScript
- make buttons, inputs, and page changes work
- organize files for a real website project
- publish a project online or prepare it to publish
- show your work to a teacher or mentor

You do **not** need to become a professional web developer.
You do need to build something real that works, and you should be able to explain part of how it works.

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">What counts as progress?</h2>

Progress in this track should happen in **small steps**.

You should not wait until the very end to prove you learned something. Each level should give you a working result.

There are three levels:
- **Bronze** - Build a basic webpage with structure and style.
- **Silver** - Add JavaScript so the page reacts to the user.
- **Gold** - Build a more complete site or app and prepare it to share.

Each level has:
- ideas to learn
- things to practice
- things to build
- a place for sign-off

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">What are HTML, CSS, and JavaScript?</h2>

These three tools work together:

- **HTML** gives the page its structure.
  - headings
  - paragraphs
  - images
  - links
  - buttons
- **CSS** changes how the page looks.
  - colors
  - fonts
  - spacing
  - borders
  - layout
- **JavaScript** makes the page do things.
  - react to clicks
  - change text
  - hide and show things
  - keep score
  - update the page while it is running

A good way to think about it:

> HTML is the skeleton, CSS is the look, and JavaScript is the behavior.

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">Setup and good habits</h2>

Some parts of this track may need help from a parent, teacher, or mentor, especially:
- creating a GitHub account
- using Git or uploading files
- publishing a site online
- installing Node or creating a React app
- using the terminal

That is normal.

Good habits for this track:
- keep your files in one clear project folder
- name files clearly
- test your page often in a browser
- save often
- change one thing at a time when debugging
- read error messages instead of pretending they do not exist
- ask for help before a tiny mistake becomes a two-hour tragedy

A very normal starter setup is:
- `index.html`
- `style.css`
- `script.js`

If your project gets bigger later, you can add folders for images, sounds, or extra pages.

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">Bronze Level</h2>

### Goal
Build a basic webpage that has real content and real styling.

### Learn
At Bronze level, you should understand these ideas:
- what HTML tags are
- how a page is structured
- how links and images work
- how to connect a CSS file to an HTML file
- how to change colors, fonts, spacing, and layout

### Practice
Try to do these things:
- make a heading and some paragraphs
- add a link
- add an image
- create a list
- change colors and fonts with CSS
- add padding, margins, or borders

### Bronze checkpoints
Complete these before asking for Bronze sign-off:
- [ ] I created an HTML page that opens in a browser.
- [ ] I used headings, paragraphs, and at least one list or image.
- [ ] I linked a CSS file to my page.
- [ ] I changed colors, spacing, or fonts with CSS.
- [ ] My page has a clear topic.
- [ ] I fixed at least one problem while testing it.
- [ ] I can show a teacher or mentor what each file is for.

### Bronze mini-project ideas
Pick at least **one**:
- an "About Me" page
- a favorite animals page
- a page about your favorite game or movie
- a page about your pet, family, or hobbies
- a fake fan page for something you like

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">Silver Level</h2>

### Goal
Add JavaScript so your webpage reacts to what the user does.

### Learn
At Silver level, you should practice these ideas:
- what variables are
- what functions are
- how buttons can trigger code
- how JavaScript can change text, styles, or content on the page
- how to read simple error messages in the browser console

### Practice
Try to do these things:
- make a button do something
- change text when someone clicks
- show or hide part of the page
- update a score, counter, or message
- take input from the user and respond to it

### Silver checkpoints
- [ ] I linked a JavaScript file to my page.
- [ ] I made a button or other input trigger code.
- [ ] My page changes in response to a click or input.
- [ ] I used at least one variable.
- [ ] I used at least one function.
- [ ] I tested my page and fixed at least one bug.
- [ ] I can explain what one part of my JavaScript does.

### Silver mini-project ideas
Pick at least **one**:
- a click counter
- a quiz page
- a random answer generator
- a light/dark mode switch
- a simple calculator
- a page that changes text or images when you click buttons

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">Gold Level</h2>

### Goal
Build a stronger website or simple web app that feels complete and organized.

### Learn
At Gold level, the big idea is this:

> A good web project is not just "I made a page." It should be clear, working, organized, and understandable to someone else.

Your project does not need to be huge. It does need to feel finished enough that another person can use it and tell what it is supposed to do.

### Build
Create your own project using one of these ideas:
- a small multi-page website
- a quiz app
- a simple game in the browser
- a recipe site
- a tracker or score app
- a fan site with multiple sections
- your own project idea

Your Gold project should include:
- HTML, CSS, and JavaScript
- a clear purpose
- at least one interactive feature
- organized files
- enough polish that another person can use it

Better version:
- use multiple pages
- use better layout and styling
- organize images in folders
- write cleaner code with useful function names
- publish the project online

### Gold checkpoints
- [ ] I built a website or web app with a clear purpose.
- [ ] My project uses HTML, CSS, and JavaScript together.
- [ ] My project includes at least one interactive feature.
- [ ] My files are organized clearly.
- [ ] I tested the project and improved it after finding problems.
- [ ] I can explain what one part of my code does.
- [ ] I showed the project to a teacher or mentor.

### Gold project examples
- a two-page website about a topic you like
- a browser quiz game
- a score tracker for a made-up game
- a page with buttons that change scenes or information
- a simple web app that keeps track of something
- your own project idea with clear interaction

<h2 style="color: #2563eb; border-bottom: 2px solid #93c5fd; padding-bottom: 0.25em;">Advanced path: Node and React</h2>

This part is **optional** and more advanced.

You do **not** need Node or React to complete the main Web Track.

But if you finish the main path and want to keep going, these are good next steps.

### Node
**Node** lets you run JavaScript outside the browser.

That means you can:
- make tools that run from the terminal
- build small servers
- organize bigger JavaScript projects

### React
**React** helps you build bigger web interfaces using components.

That means you can:
- build apps with reusable pieces
- manage larger projects more cleanly
- get closer to how many real web apps are built

### Advanced extension ideas
- publish a static site with GitHub Pages
- create a simple React app
- customize a React page and publish it
- use Node tools to help build or organize your project

This is a bonus layer, not the main requirement.
