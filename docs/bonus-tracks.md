# Coder Dojo Bonus Tracks

Welcome to the **Coder Dojo Bonus Tracks**.

These tracks are extra paths you can follow outside the main Dojo lessons. Each track helps you practice a different kind of coding by building real things step by step.

You do **not** finish a track just by reading about it. You finish a track by **making something, testing it, fixing problems, and showing your work**.

---

## What are the Bonus Tracks?

There are four Bonus Tracks:

| Track | What you learn | What you might build |
|---|---|---|
| **Data** | How to ask questions with data using SQL | A solved mystery, a small database, and useful queries |
| **Robot** | How Python can control real hardware | LED projects, buttons, alarms, and breadboard gadgets |
| **Web** | How websites and web apps work | Interactive web pages and a published site |
| **Gaming** | How video games are built | A small playable game made in Godot |

Each track has its own page with:
- clear goals
- small checkpoints
- project ideas
- required milestones
- a simple sign-off area

---

## How to use these pages

1. Start at the **main Dojo page**.
2. Open the Bonus Track page you want.
3. Work through the steps in order.
4. Save your code and projects as you go.
5. Ask for help when you get stuck.
6. Show your work to a teacher or mentor when you complete a milestone.

Some parts are easy to do on your own. Some parts may need help from a parent, teacher, or mentor, especially when installing software, setting up accounts, or wiring hardware.

---

## How progress works

Each track is built around **small wins**, not just one giant final project.

Most tracks will include three levels:

- **Bronze** - Learn the basics and finish your first small working project.
- **Silver** - Combine ideas and build something more interactive.
- **Gold** - Finish a stronger project that feels complete and that you can explain.

Each level should give you a chance to:
- learn a new idea
- practice it in a small exercise
- use it in a real mini-project

At the end of the track, there will still be a bigger final project or final check, but you should already have real progress before you get there.

---

## What counts as real progress?

These things **do count**:
- writing code yourself
- changing a tutorial project and making it your own
- fixing bugs
- building something that works
- explaining what part of your code does
- saving your work and showing it to a teacher or mentor

These things **do not count by themselves**:
- only reading a tutorial
- only watching a video
- copying code without understanding any of it
- saying you finished without showing the project

Using tutorials is fine. Real programmers use examples all the time. What matters is that you can still show what you built and explain at least part of how it works.

## Teacher or mentor sign-off

The sign-off system is meant to be simple.

When you finish a milestone or project:
1. Show your work to a teacher or mentor.
2. Explain what the project does.
3. Answer a few simple questions if needed.
4. Get a quick sign-off.

A sign-off might be used for:
- a Bronze milestone
- a Silver milestone
- a Gold milestone
- final track approval

For now, keep sign-off simple and generic.

**Example sign-off format:**

- Student name: __________________________
- Track: _________________________________
- Milestone or project: __________________
- Teacher / Mentor: ______________________
- Date: __________________________________
- Approved: Yes / No
- Notes: _________________________________

Later, this can be turned into a better printed form or a digital form.

## How coins are awarded

Coins are meant to celebrate **real completed work**.

A typical process might look like this:

1. You complete the required work for a track level or full track.
2. A teacher or mentor signs off on it.
3. The sign-off is sent back to Mr. Torr.
4. The matching coin is prepared.
5. The coin is delivered at school or at a Dojo session.

That part may change later, but the important rule stays the same:

> **No sign-off, no coin.**

That is not meant to be harsh. It just keeps the system fair.

## Choosing a track

Pick the track that sounds the most fun to you.

Here is the honest version:
- If you like solving puzzles and asking questions, start with **Data**.
- If you like lights, buttons, and real physical projects, start with **Robot**.
- If you like making websites and apps, start with **Web**.
- If you like video games and game design, start with **Gaming**.

You do **not** have to do all four tracks.

It is better to finish one track well than to half-finish four of them and end up with a pile of sadness and confusion.

---

## Good habits for every track

No matter which track you choose, these habits matter:

- Save your work often.
- Name your files clearly.
- Test your code often.
- When something breaks, change one thing at a time.
- Keep old versions if you are trying something risky.
- If you get stuck, write down what is wrong.
- Ask for help before you stay stuck forever.

That last one is not weakness. That is just avoiding unnecessary suffering.

---

## Notes for parents and teachers

Students may need adult help with:
- installing software
- creating accounts
- using the terminal
- publishing projects online
- wiring breadboards safely
- moving files between home and school

That is normal.

The goal is **not** for adults to do the work. The goal is to help students get past setup problems so they can do the important part themselves.

A good sign-off is not just "I saw a finished screen."
A good sign-off means the student can also explain:
- what the project does
- what part of the code controls it
- what problem they had to fix

## Bonus Track overview

### Data Track
Learn how tables, rows, columns, and queries work. Solve a mystery with SQL and build your own small database.

### Robot Track
Use Python to make hardware do things in the real world. Build projects with LEDs, buttons, buzzers, and sensors.

### Web Track
Start with HTML, CSS, and JavaScript. Later, move into bigger web projects and publishing your work online.

### Gaming Track
Build small games in Godot. Learn movement, rules, collisions, variables, and game logic.

## What happens next?

After reading this page, open one of the track guides and begin with the first level.

Suggested order if you are not sure:
1. **Robot** or **Data** first
2. **Web** next
3. **Gaming** whenever you are ready for a bigger project

That is only a suggestion. The best track is the one you will actually keep working on.
